# KubeFoundry 开发计划

## 一、项目概述

### 1.1 项目背景
KubeFoundry 是一个专为离线环境设计的 Kubernetes 自动化部署系统，采用控制节点-代理节点架构，通过 SSH 协议协调多个目标节点的部署工作。系统包含 15 个核心部署步骤，特别包含容器运行时版本自动判断和 kube-controller-manager 参数配置功能。

### 1.2 开发目标
- 实现完整的离线 Kubernetes 集群自动化部署
- 支持幂等执行和完善的错误处理机制
- 提供模块化、可扩展的配置管理系统
- 确保安全性、可靠性和可维护性

### 1.3 技术栈
- **开发语言**: Bash 4.4+
- **配置管理**: YAML
- **目标平台**: RHEL/CentOS/AlmaLinux/Oracle Linux 8+
- **Kubernetes 版本**: 1.23 和 1.30
- **支持架构**: x86_64 和 arm64 (可配置)

### 1.4 核心特性
- **容器运行时自动选择**: 根据 K8s 版本自动选择 Docker 或 containerd
- **kube-controller-manager 参数配置**: 自动添加集群签名期限参数
- **离线部署能力**: 完全支持离线环境部署
- **15步部署流程**: 从环境准备到备份配置的完整流程

## 五、重点功能详细说明

### 5.1 容器运行时自动选择

#### 功能描述
根据目标 Kubernetes 版本自动选择最合适的容器运行时，确保版本兼容性和最佳性能。

#### 实现逻辑
```bash
function select_container_runtime() {
    local kube_version=$1
    local runtime_config=$2

    case "${kube_version}" in
        "1.23"*)
            if [[ "${runtime_config}" == "auto" ]]; then
                echo "docker"
            else
                echo "${runtime_config}"
            fi
            ;;
        "1.30"*)
            if [[ "${runtime_config}" == "auto" ]]; then
                echo "containerd"
            else
                echo "${runtime_config}"
            fi
            ;;
        *)
            log_error "Unsupported Kubernetes version: ${kube_version}"
            return 1
            ;;
    esac
}
```

#### 配置示例
```yaml
global:
  # 系统架构，默认为 x86_64，可选值：x86_64, arm64
  architecture: x86_64

packages:
  container_runtime: auto  # auto/docker/containerd
  kube_version: 1.30.14
```

### 5.2 kube-controller-manager 参数配置

#### 功能描述
在集群初始化完成后，自动为 kube-controller-manager 添加固定的集群签名期限参数。

#### 实现步骤
1. 备份原始配置文件
2. 修改 Pod 清单，在 `spec.containers[0].command` 最后一行添加参数
3. 重启 Pod 以应用新配置
4. 验证参数生效

#### 参数内容
```yaml
- --cluster-signing-duration=867240h0m0s
```
